## 工作方法

综合报告的生成遵循以下标准化流程：

```
用户提交 3 套问卷
    │
    ├── ① 读取问卷规则（01-问卷规则/）
    │     按题目→选项→分值累加，计算各维度原始得分
    │
    ├── ② 系统精准计分（lzuScoringService.js）
    │     权重：领导风格(30%) + 人格特质(40%) + 创造力障碍(30%) = 总分100
    │     等级：卓越型(≥90) / 进取型(75-89) / 成长型(60-74) / 待发展型(<60)
    │
    ├── ③ 调用AI（发送 03-提示词/AI报告生成提示词.md）
    │     AI获得：系统分数 + 问卷题目原文 + 输出格式要求
    │     AI返回：结构化文字分析（按 [SECTION:xxx] 标记分割）
    │
    └── ④ 按模版排版（02-报告模版/综合报告排版规范.md）
          将分数+图表+AI文字按固定模块顺序组装为HTML和Word文档
```

**核心原则：**
- **提示词是唯一的调控手段**——修改报告内容只需修改提示词，不改代码
- **分数由系统计算**——AI只解读分数，不修改分数
- **模版固定化**——模块顺序、图表样式、CSS全部写死在代码中

## 修改报告的方法

如果需要调整综合报告的输出内容，按以下优先级操作：

| 调整目标 | 修改文件 | 示例 |
|---------|---------|------|
| 改变AI分析角度/语气/深度 | `03-提示词/AI报告生成提示词.md` | "把分析角度从个人发展改为人岗匹配" |
| 增加/删除报告模块 | 提示词 + 报告模版 + 代码模板 | "增加一个'沟通风格'模块" |
| 修改计分权重 | `lzuScoringService.js` + 更新本文档 | "领导力从30%调到35%" |
| 修改图表样式 | `lzuChartService.js` + 更新模版文档 | "柱状图改成横向" |
| 修改问卷题目 | JSON问卷文件 + 更新本文档 | "某道题目的选项分值调整" |
